## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing |
| md | 16px | Default element spacing, input horizontal padding |
| lg | 24px | Card internal padding, section padding |
| xl | 32px | Login card vertical padding |
| 2xl | 48px | Vertical centering offset |
| 3xl | 64px | Page-level vertical centering |

Exceptions: Touch targets on the submit button must be minimum 44px tall (accessibility). The existing `.button` class produces `py-3` (24px vertical) + `text-lg` line-height — combine to ~48px total, which satisfies the 44px minimum. No change needed.

Source: 8-point scale default. Body margin is `m-8` (32px) from `globals.css`.

## Visual Hierarchy

Primary focal point: the "Sign in" button (sole accent-colored element on the page). Visual hierarchy top-to-bottom: app name ("MTGCardSearch") → subtitle ("Enter group password to continue") → password field → "Sign in" CTA. Every other element uses foreground or foreground/opacity — the accent-filled button is the only chromatic interrupt, directing the user's eye to the single required action.

## Component Inventory

### Login Card

- Outer container: `min-h-screen flex items-center justify-center` — full-viewport centering
- Card: `bg-background border border-accent2 rounded-2xl shadow-md px-8 py-10 w-full max-w-sm`
  - `shadow-md` uses default Tailwind shadow (not accent-tinted). Clean and on-brand.
- App name heading: `text-xl font-bold font-narrow text-foreground mb-1`
  - Text: "MTGCardSearch"
- Subtitle: `text-sm text-foreground/70 mb-6`
  - Text: "Enter group password to continue"
- Password input container: full-width, label above field
  - Label: `text-sm font-bold text-foreground mb-1 block` — text: "Password"
  - Input: `w-full rounded-lg border border-accent2 bg-background px-3 py-2 text-base text-foreground placeholder:text-foreground/40 focus:outline-none focus:ring-2 focus:ring-accent1/50`
  - Placeholder: "••••••••"
- Submit button: reuse `.button` class from `globals.css` — `mt-6 w-full justify-center rounded-xl bg-accent1 px-6 py-3 text-lg font-medium text-background cursor-pointer`
  - Button text: "Sign in"
- Error message (conditional): `mt-3 text-sm font-bold text-red-600` — inline below the button, no shake animation. Appears on wrong password. Disappears on next submission attempt.
  - Copy variants: see Copywriting Contract below.

Source: CONTEXT.md decision — "Card/box layout, centered panel, single password input + submit button using existing `button` CSS class."

### Header — Logout Link

- Add "Log out" as a nav item in `header.tsx` alongside existing links
- Style: `text-sm font-bold text-foreground/60 hover:text-foreground cursor-pointer` — visually de-emphasized relative to nav links (uses opacity, not accent color)
- Position: rightmost item in the `<ul>` nav list
- Behavior: submits a POST to `/api/auth/logout` via a `<form>` with `method="post"` (or a client component button triggering a fetch). Redirects to `/login` on response.

Source: CONTEXT.md — "Logout link lives in the header nav, visible to all authenticated users."

### Admin Redirect Message

- When a group-session user hits `/admin`, proxy redirects to `/login?message=admin-required`
- Login page reads `searchParams.message` and renders a notice above the form:
  - Container: `mb-4 rounded-lg border border-accent2 bg-accent2/30 px-4 py-3`
  - Copy: "Admin access required — enter the admin password"
  - Style: `text-sm font-bold text-foreground`

Source: CONTEXT.md — "A group member hitting `/admin` is redirected to `/login` with message: 'Admin access required — enter the admin password'"
