Modify `src/lib/updateCollections.ts` to wrap the three database operations (deleteMany, createMany, user.update) inside `prisma.$transaction(async (tx) => {...})`. The scraping call (`scrapeMoxfield`) stays OUTSIDE the transaction — only database writes go inside.

Replace the per-user try/catch block (lines ~14-64) with this structure:

```typescript
try {
  const cards = await scrapeMoxfield({
    collectionId: user.moxfieldCollectionId
  });

  console.log(`Scraped ${cards.length} cards for ${user.name}`);

  if (cards.length === 0) {
    console.log('No cards scraped - skipping database update');
    continue;
  }

  console.log('Example card:', cards[0]);

  // Atomic: all three operations commit together or none do
  await prisma.$transaction(async (tx) => {
    const deleteResult = await tx.collectionCard.deleteMany({
      where: { userId: user.id }
    });
    console.log(`Deleted ${deleteResult.count} old cards`);

    const createResult = await tx.collectionCard.createMany({
      data: cards.map(card => ({
        userId: user.id,
        cardName: card.name,
        scryfallId: card.scryfall_id,
        set: card.set,
        setName: card.set_name,
        quantity: card.quantity,
        condition: card.condition,
        isFoil: card.isFoil,
        typeLine: card.type_line,
      }))
    });
    console.log(`Inserted ${createResult.count} new cards`);

    await tx.user.update({
      where: { id: user.id },
      data: { lastUpdated: new Date() }
    });
  });

  console.log(`Successfully updated ${cards.length} cards for ${user.name}`);
} catch (error) {
  // Transaction rolled back automatically — user's cards are intact
  const message = `Collection update failed for user "${user.name}" (id: ${user.id}). ` +
    `No changes were made — the user's cards are intact. ` +
    `To fix: re-trigger the collection update for this user. ` +
    `Original error: ${error instanceof Error ? error.message : String(error)}`;
  console.error(message, error);
  throw new Error(message);
}
```

Critical details:
- Use `tx.collectionCard.deleteMany` and `tx.collectionCard.createMany` — NOT `prisma.collectionCard.*` — inside the callback. Using `prisma.*` instead of `tx.*` bypasses the transaction.
- Use `tx.user.update` for lastUpdated — must be inside the transaction so it only commits on success.
- The `scrapeMoxfield()` call stays BEFORE the `$transaction` block (network I/O should not hold a transaction open).
- The catch block THROWS the error (not just logs it) so the API route returns a non-200 response. This is a locked decision from CONTEXT.md.
- Error message must include: which user failed, that cards are intact, how to fix it.
- Keep all existing `console.log` statements for the scraping portion.
- The `continue` for zero-card results stays as-is (skip transaction entirely).
