2. Rewrite `src/lib/prisma.ts` to use the Prisma 6.6.0+ adapter API. The complete file content:

   ```typescript
   import { PrismaClient } from '@prisma/client';
   import { PrismaLibSQL } from '@prisma/adapter-libsql';

   const globalForPrisma = globalThis as unknown as {
     prisma: PrismaClient | undefined;
   };

   function createPrismaClient(): PrismaClient {
     const adapter = new PrismaLibSQL({
       url: process.env.DATABASE_URL ?? '',
       authToken: process.env.DATABASE_AUTH_TOKEN,
     });
     return new PrismaClient({ adapter });
   }

   export const prisma = globalForPrisma.prisma ?? createPrismaClient();

   if (process.env.NODE_ENV !== 'production') globalForPrisma.prisma = prisma;
   ```

   Key details:
   - Do NOT use `createClient()` from `@libsql/client` — pass options directly to `PrismaLibSQL` (Prisma 6.6.0+ API)
   - `authToken` receives `process.env.DATABASE_AUTH_TOKEN` which is `undefined` locally (correct for file mode)
   - Do NOT set `authToken: ''` (empty string) — must be `undefined` for local file URLs
   - Global singleton pattern is preserved (prevents connection pool exhaustion in Next.js dev)
   - `url` falls back to `''` if DATABASE_URL is undefined (should never happen if .env is set)

3. Run `npm run build` to verify the app builds with the new adapter (no native addon errors).
