# Phase 06.1 Plan 03: Games Filter Toolbar Summary

Added an inline filter toolbar above the `/games` history table with Winner / Player count / Players multi-select controls, backed by three ephemeral `useState` hooks and a `useMemo` derived `filteredGames` list — zero new network calls, zero new dependencies, and zero references to the hidden `isImported` flag.

## What Was Built

### Pure helpers (Task 1 — TDD)

Exported from `src/app/games/page.tsx` as named exports alongside the existing `GamesPage` default export:

- `export type { Participant, Game }` — type-only re-export of existing interfaces
- `export interface FilterState { winner, playerCount, players }`
- `export function matchesAllFilters(game, filters): boolean` — D-17 AND-across / OR-within semantics. Empty filter state returns `true`.
- `export function deriveWinnerOptions(games): string[]` — D-19: distinct `participants[].playerName` where `isWinner === true`, alphabetized case-insensitive.
- `export function derivePlayerOptions(games): string[]` — D-20: distinct `participants[].playerName` across ALL participants, alphabetized case-insensitive.

Covered by `tests/games-filter.test.ts` with **20 unit tests** across 3 `describe` blocks (matchesAllFilters, deriveWinnerOptions, derivePlayerOptions).

### Filter toolbar UI (Task 2)

Inside `GamesPage` the filter state lives in three hooks placed immediately after `pendingDelete`:

```ts
const [winnerFilter, setWinnerFilter] = useState<string | null>(null);
const [countFilter, setCountFilter] = useState<2 | 3 | 4 | null>(null);
const [playerFilters, setPlayerFilters] = useState<string[]>([]);
```

Derived values (computed before `return`):

```ts
const winnerOptions = useMemo(() => deriveWinnerOptions(games), [games]);
const playerOptions = useMemo(() => derivePlayerOptions(games), [games]);
const filteredGames = useMemo(
  () => games.filter((g) => matchesAllFilters(g, { winner: winnerFilter, playerCount: countFilter, players: playerFilters })),
  [games, winnerFilter, countFilter, playerFilters]
);
const anyFilterActive = winnerFilter !== null || countFilter !== null || playerFilters.length > 0;
```

Helper callbacks `clearFilters` and `togglePlayerFilter` are defined inside the component so they close over the state setters.

The toolbar renders between the "Games / Log game" header and the error/loading/empty-state messages, guarded by `{games.length > 0 && (...)}`:

- **Winner** — native `<select>` with `"Any winner"` + one option per `winnerOptions` entry, disabled when no winners exist.
- **Player count** — native `<select>` with `"Any count"` and exactly three data options: `2 players`, `3 players`, `4 players` (no 1-player option per D-18).
- **Players** — native `<details><summary>` disclosure that opens a checkbox list sourced from `playerOptions`. Each checkbox toggles membership in `playerFilters` via `togglePlayerFilter`. The summary shows the comma-joined current selection or "Any players".
- **Clear filters** — conditionally rendered button that calls `clearFilters()` when `anyFilterActive` is true.

The table render loop was swapped from `{games.map((g) => ...)}` to `{filteredGames.map((g) => ...)}`. The outer `{games.length > 0 && ...}` guard on the `<table>` is preserved so the "No games logged yet" empty state still keys off the raw count.

A second empty-state block was added for filter-misses:

```tsx
{!isLoading && games.length > 0 && filteredGames.length === 0 && (
  <p className="text-muted">
    No games match your filters.{' '}
    <button type="button" onClick={clearFilters} className="text-accent underline">
      Clear filters
    </button>
  </p>
)}
```

## Filter combine logic (D-17)

For a game `g` and filter state `{winner, playerCount, players}`:

| Filter | When active | Pass condition |
|--------|-------------|----------------|
| `winner` | `winner !== null` | `g.participants.find(p => p.isWinner)?.playerName === winner` |
| `playerCount` | `playerCount !== null` | `g.participants.length === playerCount` |
| `players` | `players.length > 0` | `g.participants.some(p => players.includes(p.playerName))` (OR semantics) |

All active filter types are AND-ed. Empty filter state (all null / empty) returns `true` — empty = show everything.
