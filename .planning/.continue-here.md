<context>
This is a private MTG collection app for a friend group. The existing codebase works locally but can't be deployed to Vercel because: (1) better-sqlite3 requires native binaries incompatible with Vercel serverless, (2) Puppeteer needs @sparticuz/chromium-min for serverless, (3) no authentication exists on any route, (4) no automated sync. The 4 phases address these in dependency order — DB migration is the hard blocker and must go first.

Key research findings to remember:
- Vercel Hobby has 300s timeout (fluid compute enabled by default) — NOT 10s
- Next.js 16 uses `proxy.ts` not `middleware.ts` — breaking change from older patterns
- `prisma migrate deploy` doesn't work against Turso HTTP API — use `prisma db push` or `prisma migrate diff --script | turso db shell`
- Collection update in updateCollections.ts needs `$transaction` wrapping before deployment
</context>
